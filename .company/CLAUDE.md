# ROAD OCR Hub — AI 運用ルール

このファイルは、AI（Claude / Copilot）がこのプロジェクトを進める際に従うべきルールを定義する。

---

## 1. ディレクトリ構成

```
.company/
├── CLAUDE.md          # このファイル（AI 運用ルール）
├── note/              # 日々の進捗ログ
│   └── YYYY-MM-DD.md  # 日付単位のログファイル
└── todo/              # 日付ごとの TODO
    └── YYYY-MM-DD.md  # 日付単位の TODO ファイル
```

---

## 2. セッション開始時のルーティン

AI がセッションを開始したら、以下を必ず行う：

1. `.company/CLAUDE.md` を読み、ルールを確認する
2. `.company/todo/` の最新ファイルを確認し、未完了タスクを把握する
3. `.company/note/` の最新ファイルを確認し、前回の作業状況を把握する
4. 今日の日付で `todo/YYYY-MM-DD.md` が存在しなければ新規作成する
5. 前日の未完了タスクがあれば今日の TODO に引き継ぐ

---

## 3. 進捗ログ（note/）

### ファイル命名
- `note/YYYY-MM-DD.md`（例: `note/2026-04-06.md`）

### 記載内容
- その日に行った作業を時系列で箇条書きにする
- 実装した機能、修正したバグ、決定した方針を記録する
- エラーが発生した場合は原因と解決策を記録する
- ユーザーとの会話で決まった重要な方針変更を記録する

### フォーマット

```markdown
# YYYY-MM-DD 進捗ログ

## 実施内容
- [ 時刻 ] 作業内容の概要
- [ 時刻 ] 作業内容の概要

## 決定事項
- 決定した方針や変更点

## 課題・メモ
- 残った課題や気づき
```

### タイミング
- セッション中に意味のある作業を完了したタイミングで随時追記する
- セッション終了時に当日のログを最終更新する

---

## 4. TODO 管理（todo/）

### ファイル命名
- `todo/YYYY-MM-DD.md`（例: `todo/2026-04-06.md`）

### フォーマット

```markdown
# YYYY-MM-DD TODO

## 今日のタスク
- [ ] タスク内容
- [x] 完了したタスク

## 引き継ぎ（前日未完了）
- [ ] 前日から引き継いだタスク

## 次回やること
- 次のセッションでやるべきこと
```

### ルール
- タスクを完了したら即座に `[x]` に更新する
- 新しいタスクが発生したら即座に追記する
- セッション終了時に未完了タスクと「次回やること」を明記する

---

## 5. コーディング規約

### 全般
- TypeScript を使用し、`strict: true` を維持する
- コンパイルエラーが出ない状態を常に保つ

### フロントエンド（frontend/）
- Next.js 14 + React 18 + Tailwind CSS
- ダークテーマ（bg: #020617, accent: cyan-400）
- パスエイリアス `@/*` → `./*`
- コンポーネントは機能ごとに `components/機能名/` に配置

### バックエンド（backend/）
- NestJS 10 + Prisma 5 + PostgreSQL
- Module / Controller / Service / DTO のパターンを統一
- パスエイリアス `@/*` → `src/*`
- 環境変数は `@nestjs/config` の `ConfigService` 経由で取得

### データベース
- スキーマ変更は必ず Prisma Migration で行う
- シードデータは `prisma/seed.ts` で管理

---

## 6. 作業の進め方

1. **計画**: 作業前に TODO を作成し、何をやるか明確にする
2. **実装**: 1つずつタスクを進め、完了ごとに TODO を更新する
3. **検証**: コード変更後は `tsc --noEmit` でコンパイルチェックを行う
4. **記録**: 実施した内容を note に記録する
5. **報告**: ユーザーに完了した内容と次のステップを簡潔に伝える
